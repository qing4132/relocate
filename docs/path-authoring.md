# path-authoring.md — 新增 / 重写一条路径的参考工序

> 2026-04-19 更新。本文件不是硬约束——[RULES.md](../RULES.md) § 2 的"尽力而为"精神不变——但以下关于**路径必须覆盖的范围**是审美层面的共识，建议每条路径都满足。

---

## 每条路径必须覆盖的三个维度

### 维度 A · 时间纵深：工签 → 永居 → 入籍

一条"路径"不是停在工签维持，而是推到该国的终点（公民身份）。中途凡是读者真的会遇到的里程碑都要出现：

- **工签阶段** — 资格、申请、延期、维持、常见雷区
- **永居阶段** — 触发条件、时间线、维持要求（居住天数 / 税务 / 体检等）
- **入籍阶段** — 要求、宣誓、双重国籍政策、对中国护照持有者的处理

如果某国完全不允许入籍或结构上无"永居"概念（如日本的永住 vs 帰化 并行），在该段显式说明**为什么跳过 / 合并**，不要静默省略。

### 维度 B · 家庭为默认单位（不再有单独的 "family" 分支页）

每条路径页天然是家庭视角。具体要求：

- **主申的同时，配偶与子女的派生签 / 派生身份要一并写**（节点卡里用"↳ 配偶派生 / ↳ 子女派生"小块）
- **子女年龄边界必须写清**（CSPA、年满 18/21 失去派生资格、继续教育例外等）
- **"以家庭为单位的可行玩法"作为独立小节**，至少包括：
  - **双保险 / 双轨制**：两位成年人各持一条独立路径（例：一人 L-1B + 一人 L-2S 转雇主赞助 H-1B；一人 HSP + 一人独立工签；一人 EB-2 + 一人 EB-3）。写清触发条件、独立失败时如何兜底
  - **家庭三线图**：主申 / 配偶 / 子女三条时间线并列，标出依赖与独立节点
- **单人读者的读法**：在页首说明"忽略↳ 派生块即可当单人路径读"，不为单人另做一版

单人页与家庭页合并后文件命名即为 `<country>-<visa>.html`（不带 `-family` 后缀）。历史上存在的 `us-l1b-family.html` 已更名为 `us-l1b.html`。

### 维度 C · 备选与切换（"这条走不通怎么办"）

一条路径必须有一段"备选切换"，简要说明：

- **同类切换**：本签证内的其他子类（例：L-1B ↔ L-1A、EB-2 ↔ EB-3）
- **跨签证切换**：触发条件明确的横向跳（例：L-1 → H-1B 抽签 / O-1 / EB-1A；L-2 配偶 EAD → 独立 H-1B；F-1 OPT → H-1B；E-2 → EB-1C）
- **跨国切换**：主要目的国备胎（Canada Global Talent Stream / Australia Global Talent / UK Global Talent / Japan HSP / Singapore ONE Pass 等）
- **链接处理**：每个切换项给出目标路径页的**链接或占位**——
  - 目标页已存在 → 直接 `<a href="target.html">`
  - 目标页未建 → 写 `<a class="pending" title="尚未建立路径页">名称</a>` + 在页尾"待补路径"清单里登记一行

### 维度 C 补充：备选段的简要形式（当前默认）

每个备选项写成一小段，包含：**进入门槛**（一两句话）+ **切换时机**（什么时候该考虑它）+ **占位链接**。不需要完整卡片式比较表；等目标路径页都存在之后再回头做对比。

## 建议工序（5 步，顺序非强制）

### 1. 确定要写哪条路径

一句话说清："哪个国家 / 什么签证类型 / 覆盖到入籍"。不再说"单人 vs 家庭"——默认家庭。

### 2. 起骨架（抄一份现有路径）

从 `site/paths/` 下挑一份最接近的路径页复制，改 `<title>` / `header.top` / `h1` / `.caveat` / 锚点 id，保留 `<style>` 与首屏声明。

**建议段序**：
- 概述 + 家庭三线图
- 阶段 1 · 工签（主申 + ↳ 配偶 + ↳ 子女）
- 阶段 2 · 永居
- 阶段 3 · 入籍
- **家庭玩法**（双保险 / 双轨制）
- **备选切换**（同类 / 跨签证 / 跨国）
- 风险点
- 官方文件总表

### 3. 顺手把引用到的官方源记一笔

- 在 [SOURCES.md](../SOURCES.md) § 1 加一行
- 原件能下载的下到 `data/archive/<country>/<path-slug>/`
- 二手整理（律所 / 博客 / 论坛）不登记，但在页面上标"二手"

### 4. 页面上标清楚"有多确定"

- 有官方源 → 附链接
- 只有二手整理 → 段落里显式标"二手"
- 不确定 → 写"资料暂缺"，**不写具体数字**

### 5. Commit

message 写清：新增 / 修改了哪条路径的哪部分，引用了哪些源，是否新增"待补路径"登记。

## 占位链接约定

在页面上某处引用"本项目内尚未建立的路径页"时：

```html
<a class="pending" title="尚未建立路径页">Canada · Global Talent Stream</a>
```

样式建议（`.pending`）：下划线改为虚线 / 颜色偏灰，`cursor:help`。

页尾维护一个**"待补路径"清单**（可以是 HTML 注释或可见列表），记录：
- 占位出现在哪一段
- 预期的目标 slug（例：`ca-gts`、`au-gtv`、`uk-global-talent`）

建立该目标页时，全站搜索对应 class="pending" 的占位并替换为真实链接。

## 几条经验提示

- **不强求一次写完**。一条路径里只写完工签阶段先发布也行；后面补永居 / 入籍再发布
- **家庭玩法小节是最容易露怯的地方**——它暴露作者到底理解不理解这条路径。写不出"第二个成年人怎么办"，说明对这条路径的理解还停留在单人视角
- **数字少不是缺点**。一个"HSP 80 分 + 永居 1 年"的小卡片 + 一句"读懂这意味着什么"，比一堆强塞但半数不确定的字段更有用
- **写完自己读一遍**。假设自己是完全不懂这条路径的朋友，3 分钟能读懂主干吗？读不懂就改结构，不是加更多字

## 关于 pivot 前的那套严格工序

原版本（见 commit `5660347` 之前的 `docs/path-authoring.md`）曾把新增一条路径拆成 A-K 共 10 道工序，每道一个独立 commit，每个字段必配 `{value, sourceURL, fetchedAt, rawQuote, snapshotRef}` 五元组。

那套工序本身没错，适合一个"用户以此为真实决策依据"的严肃产品。当前"仅供参考、尽力而为"定位下不强制。将来若定位变回去，该文件应当回退为那个严格版本。
