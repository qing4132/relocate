# path-authoring.md — 新增一条路径的参考工序

> pivot 后（2026-04-19）重写。原先的 10 道严格工序被放宽为"建议流程"。本文件不是
> 硬约束，是经验提示——写一条新路径时可以按它走，也可以跳步，只要最终结果在
> [RULES.md](../RULES.md) § 2 的尽力而为精神里就行。

---

## 建议的 5 步

### 1. 先确定要写哪条路径

一句话说清："哪个国家 / 什么签证类型 / 覆盖到永居还是入籍 / 单人还是家庭单位"。
`design/path-detail-*.html` 里已有的样稿可作视觉节奏参考。

### 2. 把 outline 写出来

建议分段：

- **概述** — 一句话说清这条路径是什么
- **阶段 1（工签）** — 资格、申请流程、家属附签、常见分支
- **阶段 2（永居）** — 触发条件、时间线、维持要求
- **阶段 3（入籍）** — 要求、双重国籍政策、对中国护照持有者的提示
- **风险点** — 这条路径最容易踩的坑
- **官方文件** — 所引用的源

单人 vs 家庭的切换不必每条都双版本——写成"默认家庭视角 + 派生标签"，读者自己忽略
派生段即可当单人看。

### 3. 顺手把引用到的官方源记一笔

- 在 [SOURCES.md](../SOURCES.md) § 1 加一行，字段能填多少填多少
- 原件能下载的就下载，放 `data/archive/<country>/<authority>/<docId>/<YYYY-MM-DD>/`
  - 文件名建议：`<human-readable-title>.<ext>`（sha256 不再强制前缀）
  - 同目录建议附一份 `README.md` 或 `notes.md` 说明"这文件是从哪拿的、什么时候拿的"
- 二手整理（律所 / 博客 / 论坛）不登记 SOURCES.md，直接在页面上标"二手"

### 4. 在页面上标清楚"我有多确定"

- 有官方源 → 在该段落附链接（尽量点得开）
- 没官方源只有二手整理 → 段落里写"以下根据 XX 博客整理，未在官方文本中核对"
- 完全不确定 → 写"资料暂缺 / 未公开"，**不写具体数字**

### 5. Commit

- Message 写清：哪条路径新增了什么内容，引用了哪些源
- 不追求一次完美——发现错了就改，改的时候 message 说明发现过程

## 几条经验提示

- **不强求单次写完**。一条路径里只写完阶段 1 先发布也行；后面补阶段 2 / 3 再发布
- **家庭视角比单人视角信息更完整**。单人可由家庭视角"减去派生段"推得，反之成本高
- **数字少不是缺点**。一个"工签有效期 5 年、门槛 70 分"的小卡片比一堆强塞但半数不
  确定的字段更有用
- **写完自己读一遍**。假设自己是完全不懂这条路径的朋友，页面能 3 分钟读懂主干吗？
  读不懂就改结构、改用词，不是加更多字

## 关于 pivot 前的那套严格工序

原版本（见 commit `5660347` 之前的 `docs/path-authoring.md`）曾把新增一条路径拆成
A-K 共 10 道工序，每道一个独立 commit，每个字段必配 `{value, sourceURL, fetchedAt,
rawQuote, snapshotRef}` 五元组。

那套工序本身没错，适合一个"用户以此为真实决策依据"的严肃产品。但本项目定位是
"仅供参考、尽力而为"——那套严格度会把业余维护者劝退。所以不强制，不是否定。

将来本项目如果**真的**成长到"读者会据此做决定"的量级，这份本文件应该回退为那个
严格版本——那是方向性选择，不是小改。
