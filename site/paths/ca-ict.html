§ 0全景 · ICT 是什么 · 与 GTS / 普通 LMIA 的差异

ICT 是"跨国公司把现有员工从海外关联实体调到加拿大"这件事在加拿大移民法上的对应通道：以"对加拿大有显著利益"为豁免理由，免 LMIA、免 GTS 的 14 个 NOC 清单、免劳动力市场测试。处理速度上 ICT 通常 1–2 个月，比 GTS 的 4–6 周略慢，但比普通 LMIA 工签快得多。

0.1三个最常见的混淆点

ICT ≠ GTS · ICT ≠ CUSMA ICT · ICT 不是 PR

ICT vs GTS：这两条根本不是同一类工签——GTS 是 LMIA-based（雇主走 ESDC 做劳动力市场影响评估 + LMBP 承诺），用于从外面新招高技能人才；ICT 是 LMIA-exempt（IRCC 直接判 R205(a) 显著利益），用于把公司内部员工从海外调过来。同一家公司可以同时用两条通道，但同一个人通常只走其中一条。微软这种大厂的内部调动几乎一律走 ICT，因为它跳过了 GTS 必须证明"岗位独特 / 上 14 NOC 清单"的环节。

C12 通用 ICT vs CUSMA / CETA / CPTPP ICT：IRPR §205(a) 之下有四个国别代码——C12（通用 · 任何国籍）、C61（CUSMA · 美 / 墨 / 加公民）、C62（CETA · 欧盟成员国公民）、C63（CPTPP · 包括日 / 新 / 澳 / 英等多个成员国公民）。中国大陆护照持有人只能走 C12 通用档；C61–C63 在认定标准、申请文件、有效期上各有微调，但中国护照都用不上，本页之后不再细分。

ICT 是工签 · 不是 PR：这条与美国 L-1 一样——它本身只是临时居留权 + 工作权，PR 需要在加拿大本地积累 1 年工作经验后另走 Express Entry · CEC（或省提名等）通道。ICT 之所以是 PR 的天然桥梁，是因为它的累计期足够让你在加拿大攒满 CEC 的 1 年门槛。

0.4三大优势（为什么微软这种公司爱用）

免 LMIA · 配偶 OWP · 雇主端走熟路

① 免 LMIA：不必证明"加拿大本地招不到这岗位的人"——这是 GTS 与普通 LMIA 都绕不开的关；ICT 直接用"公司内部调动对加拿大有显著利益"做豁免理由。对一家在加拿大已有几千员工的微软来说，这个豁免几乎是默认状态。

② 配偶 OWP 免谈：同 GTS 一样，主申家属里的配偶 / 同居伴侣自动拿 Open Work Permit · 不挑雇主、不挑职业、不挑工资。对夫妻双方都在中国 IT / 金融行业的家庭，这是把 ICT 主申变成"全家都能在加正常工作 + 双线攒 CEC"的核心机制。详见 § 5。

③ 雇主端流程极熟：微软 / Google / Amazon 这类公司每年从全球各地调几百到几千人到加拿大，企业法务和移民律所对 ICT 文件包已经做到流水线级别。对申请人本人意味着：你几乎不需要自己理解三档定义的细节——公司会按"你的 title 在内部 leveling 体系里对应哪一档"自动配律所文件，你只负责确认信息无误、按时去 VAC 录指纹、按时入境。

§ 1ICT 工签 · 三档 + 关系 + 1 年硬要件

ICT 不是按"岗位 title"判，是按"实际工作内容是否符合 Executive / Senior Manager / Specialized Knowledge 三档之一的法定描述"判。这一档定下来，后面的工签有效期、累计期上限、续签难度、是否能转 PR 都跟着走。

1.1三档定义 · 落档决定一切

§1.1 · IRPR §205(a) · C12 · Executive
高管档 · Executive

"主导（direct）组织或其主要组成部分的管理；在政策制定上行使广泛裁量权；仅接受董事会 / 股东会 / 更高级别高管的一般监督。"

典型岗位：加拿大子公司 GM / Country Head · CTO / CFO · VP 级以上的全球职能负责人。

累计期上限：7 年。首签常给 2–3 年，最长可续到累计 7 年。

工资底线：与加拿大同级别可比岗位一致，无强制 prevailing wage 测试（但实操中 IRCC 仍会看薪资是否合理）。

§1.1 · IRPR §205(a) · C12 · Senior Manager
高级管理者档 · Senior Manager

"管理组织 / 部门 / 分部 / 职能部门；监督并控制其他主管 / 专业雇员 / 经理；或负责组织的某一必要职能（essential function）；在日常决策上行使裁量权。"

关键细化："function manager"（职能经理）是 2010 年后被 IRCC 明确纳入这一档的——你不必直接管人，但必须负责一个对组织运作必要的职能（如负责某条产品线、某个 SRE 体系、某个合规职能）。

典型岗位：Engineering Manager / Director / Principal Manager · Tech Lead 级且承担岗位职责定义 + 招聘权 · 大厂"M2 / M3"以上层级。

累计期上限：7 年（与 Executive 同档）。

工资底线：与同级别可比岗位一致。

§1.1 · IRPR §205(a) · C12 · Specialized Knowledge（SK）
专门知识档 · Specialized Knowledge

"两要素必须同时满足"——这是 2014 年 IRCC 改革后明文化的最大变化：

  • (A) Advanced expertise：对公司的产品 / 服务 / 研究 / 设备 / 技术 / 管理方式有同行业中不易复制的高级知识。
  • (B) Proprietary knowledge：对组织内专有的流程 / 程序 / 内部系统有专属知识——这一项把许多"虽然技术资深、但用的都是行业通用栈"的工程师挡在外面。

典型岗位：开发 / 维护公司专有大型系统的资深工程师（如 Azure 内部基础设施、Google Borg、Amazon Dynamo 团队的资深成员）· 公司专有算法 / 模型 / 数据管线的核心贡献者 · 内部专属工具链的主导设计者。

反例（IRCC 常拒）：纯前端 / 通用全栈 SDE · 用的全是公开技术栈（React / Kubernetes / PostgreSQL）的开发者 · title 是 Senior 但实际职责是按需求做 ticket。这些岗位即使工资高，也大概率因不满足 (B) Proprietary knowledge 被拒。

累计期上限：5 年。首签常给 1–3 年，最长可续到累计 5 年。

工资底线：必须达到 ESDC prevailing wage——这是 2014 改革的另一硬要求，意在防止"以 SK 名义低薪雇佣"。岗位 NOC 的本省工资中位数低于此线即拒。

1.2"Qualifying relationship" · 公司之间的关系认定

海外原雇主与加方接收实体必须是同一集团

不是任意两家"合作公司"都能用 ICT——IRCC 认可的关系限于以下四种：

  • Parent（母公司）：加方实体直接持有原雇主多数股权
  • Subsidiary（子公司）：原雇主直接持有加方实体多数股权
  • Branch（分支机构）：同一法人在两国的不同运营点（同一法人主体，分支注册）
  • Affiliate（关联实体）：两边由同一母公司或同一组人控股，且关系真实有效

对微软 / Google / Amazon 等：原雇主（如 Microsoft Mobile Technologies Beijing 或 Microsoft (China) Co., Ltd.）与接收方（Microsoft Canada Co.）都是同一全球总部（Microsoft Corporation, Redmond）下的子公司——典型的 Affiliate 关系，文件层面非常清晰。

潜在坑：同集团内的咨询公司 / 派遣公司 / 联营企业 / 合资公司 / 加盟商未必构成 qualifying relationship · 需逐案看股权与控制结构。被外包给微软中国 / 联想等"派往微软"的人员，调动时大概率走不通。

1.3"过去 3 年内连续 ≥ 1 年" 的硬卡点

在原雇主的连续全职工作经历

IRCC 要求申请人在递 ICT 申请之前 3 年内，为海外原雇主（或其 qualifying related entity）连续全职工作累计 ≥ 1 年。这一年是 ICT 路径上最容易卡的硬条件。

什么算"连续"：同一职位 / 同一雇主不间断 12 个月，正常带薪假与公司休假不打断。

什么不算：

  • 实习 / 派遣 / 外包合同——必须是该公司直接雇佣
  • 兼职 / 顾问——必须是全职
  • 读书 / Gap / 长期病假超过几个月——会被认为不连续
  • 跨集团内不同实体之间频繁调动且每段都 < 12 个月——也可能被认为不连续

"过去 3 年内"如何理解：不是"刚好 12 个月前还在该公司"，而是"递申请前 3 年的任意一段连续 12 个月"。换句话说：若现在已离职在家半年，但离职前在原雇主连续干了 2 年，仍然合格——只要 3 年窗口未越过。

对你（北京出发）的实操：如果配偶现在已经在微软中国连续干满 1 年，这一项无忧；如果是中途从其他公司跳进微软，必须等到入职满 12 个月才能启动 ICT 申请。

1.5"新办公室"变体 · 加方实体尚未运营满 1 年

New Office ICT · 首签限 1 年 · 适用极少数派人开张场景

若加方接收实体在加运营尚不足 1 年（如刚注册、刚租办公室），ICT 可以走"new office"变体——但首签限 1 年，且必须提交：

  • 加方实体的运营计划 · 一年内的雇佣规模 + 财务预测
  • 已购买 / 租赁的实际办公场地证明
  • 母公司对加方实体的资金注入计划

1 年后续签时，IRCC 会按"是否真已运营 + 真已雇人"重新审。对从北京调微软的场景这一变体几乎用不上——微软加拿大公司（Microsoft Canada Co.）早在 1985 年就成立，运营 40 余年。

1.6家属派生

§1.6 · IMP · C41 Spousal OWP + 子女 SP
配偶自动 OWP · 22 岁以下子女可读公立 K-12 免 SP
  • 配偶 / 同居伴侣：同名递 Open Work Permit（LMIA-exempt 类 · 历史上是 C41，2024 年后部分细分至 C46 等）· 有效期与主申 WP 同步 · 任何雇主任何岗位均可工作
  • 22 岁以下未婚子女：K-12 阶段公立学校免 Study Permit可直接入学；大专 / 大学阶段须申 SP · 子女可在 SP 持证期间打工 ≤ 20 h/周
  • 同性伴侣：加拿大承认 · 与异性婚姻同等待遇
  • 父母 / 兄弟姐妹：不能作为 ICT 派生家属随同；只能后续走探亲或父母 Super Visa（10 年多次往返）

对夫妻双方都在中国职业线上的家庭——这是 ICT 路径上对你最重要的一条规则。配偶 OWP 不挑职业 / 不挑薪资 / 不挑雇主，意味着你（作为配偶）落地加拿大后可以：

  • 继续给中国的雇主做远程（注意中加无双边社保协定 + 你已是加拿大税务居民，要按全球收入报税）
  • 找加拿大本地雇主（同行业 IT 工程师就业市场温哥华 / 多伦多 / 蒙特利尔都活跃）
  • 自办公司 / 自雇 / 顾问，给自己发 T4 工资以攒 CEC（详见 § 5.2）

§ 2入境与维持 · 累计期 · 续签 · BOWP · 税

ICT 工签的累计期上限是这条路径最独特的硬约束：SK 5 年、Mgr / Exec 7 年。这一上限直接决定了"在累计期归零前你必须拿到 PR 或主动出境 1 年重置"。所幸 1 年 CEC 经验门槛远低于这两个上限，正常推进基本不会被卡。

2.1首签与续签 · "recapture" 算天数

首签 1–3 年 · 累计期内可不断续签 · "recapture"机制可追回出境天数

首签长度：SK 档常给 1–3 年；Mgr / Exec 常给 2–3 年。具体由 IRCC 看 offer 长度 + 申请人申请时长 + 公司声誉决定。

续签：累计期未满前可不断续，每次续 1–3 年。续签时若岗位升级（如从 SK 升 Mgr），累计期上限也会跟着改用更高档（7 年）。

"Recapture"（追回）：累计期上限的"5 年 / 7 年"指的是在加拿大物理存在的累计时间——你出境的天数（出差 / 探亲 / 度假 / 短期回国）可以申请追回到剩余额度。这对频繁跨境出差的人是关键缓冲。

累计期满后：必须在加拿大境外为原集团关联实体连续工作 ≥ 1 年，才能重新启动一次 ICT 周期。实务里几乎没人走到这一步——因为 1 年内基本都已经在走 PR。

2.2BOWP · 递 PR 后的工签续命

引用 · ca-gts § 2.2 已递完整 Express Entry PR 申请、当前 WP 剩 ≤ 4 个月、收到 R1（Acknowledgement of Receipt）即可申 BOWP · 一次性 24 个月开放工签。BOWP 是开放工签——拿到那一刻主申自动解除雇主绑定。详见 ca-gts § 2.2。
ICT 专属：BOWP 一旦签发，原 ICT 累计期上限即不再约束（已转为 open work permit · 不属于 R205(a) 范畴）；这是 SK 档申请人在 5 年累计期临近时的关键保险阀。

2.3持证期间的可做与禁忌

  • 同雇主内调岗：同一雇主内的横向调动（同档内 SK→SK 或 Mgr→Mgr）通常可凭新岗位职责描述更新备注；跨档（SK→Mgr）须重申 WP
  • 换雇主：ICT 是雇主绑定的封闭工签——换雇主须新雇主走另一份 LMIA / LMIA-exempt / GTS / ICT 申请，原 ICT 累计期不带过去
  • 副业 / 自雇 / 顾问：原则上不允许（雇主绑定）；想做副业只能等拿到 BOWP 或 PR
  • 出境旅行：持 WP + 护照 + 有效 TRV 即可返加；如 TRV 过期须在境外重申才能登机

2.4税务 · CRA

引用 · ca-gts § 2.3 入境当日起若建立居住纽带即成为加拿大税务居民 · 按全球收入报 T1 · 联邦 + 省两级累进（Ontario 顶档 ~ 53% · Alberta ~ 48% · BC ~ 53.5%）· 工签持有人与公民 / PR 同样缴 CPP / EI · 中加有避免双重征税的 DTA。详见 ca-gts § 2.3。
ICT 北京出发专属补充：
  • 中加无双边社保协定。中国"五险一金"在出境后停缴，CPP / EI 从零开始攒，互不相通。
  • 中国境内股权 / 房产 / 理财，若入境当年市值 ≥ CAD 100,000，须在 T1 申报附 T1135 表（Foreign Income Verification Statement）· 不申报罚款重。
  • 原微软中国的 RSU / 期权，登陆前已 vested 的部分通常视作"入境前已实现收入"，登陆后再 vest 的部分按加拿大税法处理 · 跨境部分常见用 sourcing rule 拆分 · 律所 / 税务师通常会主导这一项，但你要确认。

§ 3升 PR · Express Entry · CEC 为主

ICT 持证者升 PR 的路径与 GTS 持证者完全一致——都是走 Express Entry · CEC 通道：在加全职工作满 1 年（NOC TEER 0/1/2/3）→ 入 EE pool → CRS 打分跨 cutoff → 收 ITA → 60 天内递 e-APR → 6 个月审 → COPR → 登陆 PR。

引用 · ca-gts § 3 Express Entry 三个门类（CEC / FSW / FST）的对比、CRS 六大因子打分（年龄 / 学历 / 语言 / 加拿大经验 + 配偶 + 可迁移技能 + Additional Points）、ITA → COPR 6 个月时间线、PR 维持 5/730 物理存在规则——与 GTS 完全一致，不在此重复。详见 ca-gts § 3 全段。

3.1ICT 持证者的 CRS 优势点

  • 1 年加拿大经验：ICT 工签满 1 年即攒到 CEC 最低要求 · 也是 CRS"加拿大工作经验"分项 (40–80 分)的起点
  • 雇主 long-term offer：若加方雇主愿出具长期 offer + 主申岗位为 TEER 0/1，CRS Additional Points +50（TEER 1）或 +200（特定 NOC 0 高管）—— ICT 持证者通常天然满足"已被雇主长期雇佣"的事实
  • SK 档对应 NOC 通常是 0/1：Engineering Manager / Director 类岗位天然落 NOC 0；Senior SDE 落 NOC 21231/21232（TEER 1）—— 都属于 CEC 资格内
  • STEM category-based draw：2023 起 IRCC 频繁开 STEM 专项 draw · CRS cutoff 显著低于 general（曾跨 430–480）· 微软调入的工程师 / data scientist / engineering manager 几乎全部落在 STEM 类

3.2ICT 持证者的 CRS 软肋（如有）

  • 无加拿大学历：从中国直接调入的人不会有加学历 +15/+30 分；可通过配偶在加修一年制研究生（少数学校认 1 年硕士 +15）补
  • 法语：从中国 IT 行业调过来的人法语基本为零 · 失去 2023 后的 +25/+50 法语加分；蒙特利尔 / 渥太华岗位的同事若有法语优势会显著高分
  • 年龄超过 30：年龄分从 30 岁开始每年掉 5 分（核心因子 single：30 岁 105 分 → 35 岁 76 分）· 调动时机越早越有利

§ 4升公民 · 3/5 物理存在 · 双重国籍冲突

PR 之后的入籍环节，ICT 路径与 GTS 路径走的是同一套 Citizenship Act 流程。

引用 · ca-gts § 4 公民法 §5 五条门槛（18 岁 / 5 年内 1,095 天物理存在 / 5 年内 3 次报税 / 入籍考试 / 18–54 岁 CLB 4 语言）· 递申请 → 考试 → 宣誓 · 12–18 个月处理 · 未成年子女随父母同批宣誓免天数免考试——与 GTS 完全一致。详见 ca-gts § 4 全段。
中国国籍法 §9 触发点也完全一致："定居外国的中国公民，自愿加入或取得外国国籍的，自动丧失中国国籍。" 加入加籍那一刻中国籍自动失效 · 中国护照须注销 · 之后回中国须持加拿大护照 + 中国签证。

4.1ICT 持证者的入籍时间线推算

1,095 天的"工签期半值计入"对 ICT 持证者尤其友好——这是一项很多人忽略的规则：

  • 申请前 5 年内，持 PR 期间的每一天 = 1 天
  • 申请前 5 年内，持合法临时身份（含 ICT 工签）的每一天 = 0.5 天，最多 365 天

典型推算（ICT 1 年 → PR 之间无 gap）：

  • Y0 入境（ICT 起算）→ Y1 达 CEC → Y1.5 递 EE → Y2 ITA 后递 PR → Y2.5 PR 登陆
  • 从 Y0 到 Y2.5 在工签期累计 ~ 2.5 年 × 0.5 = 1.25 年（封顶 1 年）
  • 从 Y2.5 起以 PR 身份累计满 ~ 2 年（1,095 − 365 ≈ 730 天）即可申公民
  • 总时间线：Y0 ICT 入境 → Y4.5 申公民 → Y5–Y6 宣誓拿护照

这是从中国大陆出境到拿加拿大护照较快的家庭组合之一（与 GTS 节奏相近）。

§ 5家庭玩法 · 双线 / 双轨 / 备份

ICT + 配偶 OWP 的组合，在"夫妻都在中国从事 IT / 金融 / 工程类专业工作"的家庭场景里，几乎是把"一方拿到 offer 即全家落地 + 双线攒 CEC"做到极致的玩法。

5.1双线 A · 主申 ICT + 配偶 OWP（最常见）

配偶自由就业 · 1 份 PR 申请全家派生

最经典配置：主申 ICT · 配偶 OWP 立即就业 · 子女 K-12 免 SP。第 2 年主申递 Express Entry（CEC）· 1 份 PR 申请覆盖全家。

对你的家庭场景：配偶通过 MSFT 内部调动持 ICT WP 入境 · 你以配偶身份拿 OWP · 子女（若有）入公立 K-12 免 SP。整套流程的 immigration counsel 由微软指定的律所主导。

关键提示：你拿 OWP 后在加拿大本地找工作 / 自雇 / 自办公司给自己发 T4 · 若工作 NOC 落在 TEER 0/1/2/3 · 1 年后你也满足 CEC——这是双线 B 的基础。

5.2双线 B · 夫妻分别建 Express Entry 档案

两方都满足 CEC · 两条 PR 线独立并行

若夫妻双方都持 WP（一方 ICT、另一方 OWP）且都满足 CEC 最低要求（1 年加拿大全职工作 NOC TEER 0/1/2/3 + 英语 CLB 7），分别建立 Express Entry 档案：

  • 谁的 CRS 先跨当期 cutoff，谁先收 ITA
  • 另一方作为 accompanying spouse 同批过 · 不掉队
  • 主申的反向选择带来 CRS spouse factor 调换的可能
  • category-based draws（STEM / Healthcare / Trade）对岗位 NOC 敏感——双线覆盖更多机会

对你的家庭场景的实意：配偶走 ICT · 你拿 OWP 后在本地工作 1 年（无论是给加拿大公司、给微软远程、还是自办公司给自己发 T4）。1 年后两人都能进 EE pool。若你的本地岗位恰好属于 STEM NOC（21230 / 21231 / 21311 等），你那条线在 STEM category-based draw 里可能比配偶（如管理类 NOC 0）跑得还快。

5.3三线 C · ICT + 省提名（PNP）作为 CRS cutoff 备份

CRS 卡 cutoff 时走省提名兜底 · +600 几乎锁 ITA

Ontario Immigrant Nominee Program（OINP）· BC PNP（Tech Stream）· Alberta AAIP 等多省都有 Tech 专项配合 Express Entry。多数 PNP 要求申请人在该省持有 job offer + 在该省居住意愿。

对 ICT 持证者的实操：若主申落地 Toronto（OINP 覆盖）或 Vancouver（BC PNP 覆盖），且 CRS 自评只有 460–480 略低于 general cutoff · 可同步递省提名作 plan B。被提名后 CRS +600 几乎必拿 ITA。

代价：部分 PNP 要求落地后承诺在该省居住一段时间——道德义务而非法律强制，但若拿 PR 后立刻迁出可能影响后续 PR 卡续。

§ 7核心风险点

ICT 申请 / 续签的结构性风险

  • SK 档审查 2024 年起持续趋严：IRCC 越来越严格地把"高级 + 专有"两要素拆开判 · 用通用技术栈的资深工程师 / 全栈 SDE / 通用 SRE 即使薪资达 prevailing wage 也常被以"未满足 Proprietary Knowledge"拒批。微软等大厂会用 RSU vesting schedule / 内部专有系统贡献 git history 等证据加固，但仍有不确定性。建议优先看能否走 Senior Manager 档（function manager 解释空间更大）。
  • 累计期上限：SK 5 年 / Mgr-Exec 7 年。临近上限须已拿 BOWP 或 PR，否则得出境 1 年才能重启。实务中走到上限的少，但若中途 PR 申请被拒走二审则风险上升。
  • "过去 3 年 ≥ 1 年"硬条件：若刚跳到微软不久、未满 12 个月即想调动，技术上不合格——只能等到入职满 1 年再走。
  • Qualifying relationship 文件层风险：大厂内部一般无碍 · 但 JV / 子公司股权变更 / 集团重组 / 业务剥离时可能瞬时打破 qualifying relationship · 申请人的 ICT 资格随之失效。
  • VAC 中国排队周期波动：北京 / 上海 / 广州 VAC 在签证旺季（5–8 月）排队可能拉长 · 整体处理时间从声称的"数周"延到 2–3 个月。

Express Entry / PR 阶段的风险（与 GTS 共有）

  • CRS cutoff 上升：2023 Q1 一般 draw 跨 500 分高峰 526；ICT 持证者若年龄已 35+ 且无加学历 / 无法语，CRS 自评可能落在 450–480 区间，仅靠 general draw 拿 ITA 风险大——必须依赖 STEM / Healthcare / French category-based draw 或省提名兜底。
  • category-based draws 不覆盖所有 NOC：STEM / Healthcare / French / Trade / Agriculture / Transport 六个类别外的 NOC 只能等 general draw。
  • Medical / Criminal 背调：既往健康问题（慢性病 + 超额医保预期）或其他国家刑事记录会 RFE 甚至直接拒批。

PR 维持与入籍阶段（与 GTS 共有 · 北京特别提醒）

  • Residency Obligation：5 年内不足 730 天物理存在 → PR 可撤销。最常见坑：PR 登陆后被公司调回中国长期工作 · 即使被加拿大雇主派驻可计 · 实务上须有明确派驻文件。
  • 跨境社保切割：中加无双边社保协定 · 在加期间不能继续向中国"五险一金"缴费；CPP / EI 从零开始攒 · 退休前回中国前需评估两边养老金衔接。
  • 子女兵役 / 学籍同步：加拿大无强制兵役（与新加坡不同），但若子女已是中国户籍 · 父母失籍后子女户籍 / 学籍需同步处理。
  • 宣誓前的中国公证瓶颈：中国领馆办理出生证 / 结婚证公证近年趋严 · 部分申请人因 PR 期间无法回国 / 中国公证滞后影响宣誓时间。
